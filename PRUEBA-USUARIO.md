# Protocolo de prueba con usuario real — Mi Diario Parkinson

> Lo más importante que falta. Ningún análisis de código dice si la app sirve para TI.
> Esto lo responde en una tarde. No es un examen: puedes parar cuando quieras.

## Antes de empezar
- Úsala en **tu teléfono real**, con internet (la voz lo necesita).
- Hazlo **con un acompañante** que anote (tú no tienes que escribir nada).
- Haz la prueba **dos veces el mismo día**: una en estado **ON** (buen momento) y otra en estado **OFF** (peor momento). Eso es la mitad de la verdad de esta app.
- Sesiones cortas (10–15 min). Si te cansas, para.

---

## Parte A — Tareas reales (¿puede hacer lo básico?)

Para cada tarea: el acompañante marca ✅ si la lograste, anota cuántos **intentos** y una **molestia 1–5** (1 fácil, 5 agotador).

| # | Tarea | Cómo | ✅ | Intentos | Molestia 1–5 |
|---|-------|------|----|----------|--------------|
| 1 | Registrar una toma | Por voz: "tomé la pastilla" | | | |
| 2 | Registrar una toma | Por botón: 💊 Tomé remedio | | | |
| 3 | Marcar que estás mal | Voz: "estoy en off" / botón 🟠 | | | |
| 4 | Marcar un síntoma | Voz: "temblor" / botón | | | |
| 5 | Ver tu próxima toma | Mirar la tarjeta de arriba | | | |
| 6 | Borrar un registro y deshacer | Tocar ×, luego "Deshacer" | | | |
| 7 | Abrir el Reporte | Pestaña 📊 | | | |
| 8 | Generar el PDF para el médico | Botón "Descargar PDF" | | | |

**Criterio:** si una tarea te cuesta 3+ intentos o molestia 4–5, ahí hay algo que arreglar.

---

## Parte B — Prueba de VOZ (la medición clave)

Di estas 20 frases **a tu volumen y ritmo normales** (no fuerces la voz). El acompañante marca, por cada una, si la app la entendió y registró lo correcto.

Frases:
1. tomé la pastilla
2. tomé Prolopa
3. estoy en off
4. me siento on
5. temblor
6. se me pegan los pies
7. ánimo bajo
8. dormí mal
9. me caí
10. mareo al pararme
11. estoy rígido
12. tengo dolor
13. tomé el remedio
14. estoy mal
15. estoy bien
16. me tiembla la mano
17. vi algo que no estaba
18. estoy bloqueado
19. tomé Elbrus
20. me siento lento

| Resultado | Cuántas |
|-----------|---------|
| ✅ Entendió y registró lo correcto | |
| ⚠️ Entendió algo, pero registró otra cosa | |
| ❌ No entendió / "no te escuché" | |

**Cálculo:** % de acierto = (✅ ÷ 20) × 100.
- **≥ 90%** → la voz te sirve; seguimos afinando detalles.
- **70–89%** → usable, pero conviene apoyarse en botones.
- **< 70%** → la voz no es confiable para tu voz hoy; **los botones pasan a ser la vía principal** y la voz queda como extra.

Repite 3 frases en **modo avión** (sin internet): deberían fallar con el aviso "la voz necesita internet". Eso confirma que el aviso funciona.

---

## Parte C — Solo botones (día OFF, sin voz)

En estado OFF, repite las tareas 1–8 **sin usar la voz**, solo tocando botones.
- ¿Llegas a todos los botones con una sola mano?
- ¿Tocas el botón equivocado por temblor? Anota cuántas veces.
- ¿El texto se lee bien?

**Criterio:** si puedes registrar toma + síntoma + ver próxima toma solo con botones en OFF, la app cumple su promesa básica aunque la voz falle.

---

## Qué hacemos con los resultados
Pásame la tabla llena (puedes dictármela) y con esos números:
- Ajusto el tiempo de pausa de la voz, los botones, los tamaños, lo que haga falta.
- Decidimos si la voz es vía principal o apoyo.
- Esos datos reales son, además, la mejor prueba para mostrarle a un inversor o a un fondo: "probado con un paciente, esto es lo que mejoró".
